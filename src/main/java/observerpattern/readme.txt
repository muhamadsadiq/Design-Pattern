the Observer pattern is a softwere design pattern in whitch an object, 
called the subject, maintains a list if its dependents, called observers, 
and notifies them automatically of any state changes, 
usually by calling one of their methods



when yo use Observer Pattern
⦁	Loose coupling is a benifit
⦁	The Subject (publisher) doesn't need to know anything about the Observers(subscribers)
⦁	Negatives: The Subject (publisher) may send updates that don't matter to the Observer (subscriber)
